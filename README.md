## Development Blog
* v26.4.5 : "Timestamp display now includes day name : Updated all timestamp displays to use toLocaleDateString with options for weekday, providing format like "Mon, Dec 25, 2023, 10:30 AM"; Calendar view improved: Added day-of-week headers (Sun-Sat) and proper day positioning. Days now start on the correct weekday instead of always starting with "1" in the top-left corner, matching a real calendar layout; The clear button has been added to the search bar. It will appear as an "❌" button next to the search input whenever there's text entered, and disappear when the input is empty. Clicking the button clears the search text and refreshes the tweet feed to show all tweets; when you click on a calendar day to filter tweets by date, the clear button (❌) will appear next to the search input, allowing you to clear the date filter; The calendar view will now automatically minimize (hide) after clicking on a date to filter the tweets."
* v26.3.28 : "In this update, we’ve introduced several new features and squashed a few bugs to improve your experience. You can now edit your tweets without altering their original timestamps, and we've added the convenience of pasting images directly into the tweet box, which functions just like a standard upload. For media viewing, we've enabled swipe navigation so you can easily switch between images. We've also refined how threads work: replying within the thread view now attaches your response directly to the root tweet, and clicking "thread" on any deeply nested reply will now correctly fetch the entire conversation from the very top. Finally, we've resolved an issue to ensure the date filtering feature works exactly as intended."
